BACKGROUND

プロジェクトの背景

オープンキャンパス運営では、多くの情報をリアルタイムで共有する必要があります。 Miracas Management Systemは、その課題を解決するために開発しました。

なぜ開発したのか

オープンキャンパス当日は、来場者受付・教室案内・ 人員配置・チーム連携など、多くの業務が同時に進行します。

従来は紙や口頭で情報共有を行う場面が多く、 情報更新の遅れや確認漏れが発生しやすいという課題がありました。

そこで、必要な情報を一つのシステムで管理し、 スタッフ全員がリアルタイムで同じ情報を確認できる環境を目指しました。

解決したかった課題

  • 来場者情報をリアルタイムで共有したい
  • 受付業務を効率化したい
  • 人員配置を分かりやすく管理したい
  • チーム間の情報共有を改善したい
  • オープンキャンパス全体を一元管理したい
CHALLENGES

解決すべき課題

オープンキャンパス運営では、受付だけでなく情報共有や人員配置など、 多くの業務を効率よく進める必要がありました。

01

来場者管理の効率化

来場者情報をリアルタイムで管理し、 検索・受付・教室振り分けまで 一つのシステムで完結できるようにしました。

02

情報共有の改善

チャット機能やリアルタイム同期を活用し、 スタッフ全員が最新情報を すぐ確認できる環境を構築しました。

03

人員配置の最適化

ドラッグ＆ドロップによる配置変更や 欠席管理を行えるようにし、 当日の運営負担を軽減しました。

04

オープンキャンパス全体の一元管理

イベント管理・AI要約・PDF出力など、 運営に必要な機能を一つに集約し、 管理業務を効率化しました。

SYSTEM FEATURES

システム機能

Miracas Management Systemでは、 オープンキャンパス運営を支えるための8つの主要機能を実装しました。

01

ホーム画面

システム全体の情報を確認するトップページ。 最新連絡・イベント情報・目標・ショートカットを表示し、 必要な機能へ素早くアクセスできます。

02

座席表管理

ドラッグ＆ドロップで座席を配置し、 Firestoreへ保存。 当日の座席状況をリアルタイムで管理できます。

03

来場者管理

来場者一覧・検索・受付・教室振り分け・ ステータス管理を一つの画面で行えるようにしました。

04

人員配置図

各チームの配置をドラッグ＆ドロップで変更。 欠席管理やリアルタイム同期にも対応しています。

05

チャット

リアルタイムチャット・通知・リアクション・ 未読管理など、 スタッフ同士の情報共有を支援します。

06

OC管理

FullCalendarを利用したイベント管理、 AI要約・PDF出力・Google Sheets連携など、 オープンキャンパス運営を効率化します。

07

ルーレット

メンバーやチームをランダムで抽選できる機能を実装し、 公平な担当決めをサポートします。

08

システム設定

管理者設定・利用時間・ページ公開設定・ ログ管理・チャット管理など、 システム全体を管理できる機能を実装しました。

TECH STACK

使用技術

Miracas Management Systemでは、 フロントエンドから認証・データベース・AI連携まで、 複数の技術を組み合わせてシステムを構築しました。

Frontend

HTML5 / CSS3 / JavaScript

システム全体の画面実装を担当。 レスポンシブ対応やUIアニメーションも実装しました。

UI Library

FullCalendar

オープンキャンパスの日程管理や イベントスケジュールをカレンダー形式で表示しました。

Database

Firebase Firestore

来場者情報・座席表・人員配置・ チャットデータなどをリアルタイムで管理しました。

Authentication

Firebase Authentication

スタッフ・管理者のログイン認証を実装し、 権限に応じた画面制御を行いました。

Hosting

Firebase Hosting

Webアプリケーションを公開し、 ブラウザから利用できる環境を構築しました。

AI

Cloudflare Workers AI

オープンキャンパス終了後の振り返り内容を AIで要約する機能を実装しました。

Google Workspace

Google Apps Script / Google Sheets

PDF出力やスプレッドシートとの連携を行い、 運営資料を自動生成できるようにしました。

Development

GitHub / VS Code

GitHubでソースコードを管理し、 VS Codeを使用してチーム開発を行いました。
